REKAYASA PERANGKAT LUNAK (RPL) - 19 JANUARI 2018
DEFINISI PERANGKAT LUNAK :
SUATU KUMPULAN INSTRUKSI DARI BEBERAPA PROGRAM KOMPUTER YANG APABILA DI EKSEKUSI AKAN MENYEDIAKAN BEBERAPA FUNGSI DAN PEKERJAAN YANG DI INGINKAN.

KARAKTERISTIK PERANGKAT LUNAK
- MUDAH DIRAWAT >> DILENGKAPI DOKUMENTASI DAN PERUBAHAN DAPAT DI LAKUKAN DENGAN BIAYA MINIMUM
- DAPAT DIANDALKAN >> BEKERJA SEPERTI YANG DI HARAPKAN DAN GAGAL HANYA BILA KELUAR DARI SPESIFIKASI
- EFEKTIF >> TUJUAN TERCAPAI
- EFISIEN >> EFEKTIF YANG CEPAT

PEKERJAAN RPL :
- KEBUTUHAN USER (SYSTEM ANALYST)
- BENTUK RANCANGAN (SYSTEM ANALYST)
- BAHASA KOMPUTER (PROGRAMMER)
- KODE MESIN (PROGRAMMER)

RPL MENGHASILKAN :
- KOMPONEN PL YANG DAPAT DI PAKAI ULANG
- REUSABLE COMPONENT
   SUBROUTINE : BERISI ALGORITMA
   OBJECT CLASS : BERISI ALGORITMA & STRUKTUR DATA
- PRODUK PL DOKUMENTASI HARUS LENGKAP
  HARUS JELAS, BATASAN SISTEM HARUS JELAS, CLEAR DARI AWAL.

PRODUK RPL :
- GENERIK : PRODUK DI KEMBANGKAN UNTUK DI JUAL KE PUBLIK
- SPESIFIK : PRODUK DI KEMBANGKAN KHUSUS UNTUK SEBUAH PERUSAHAAN

JENIS PRODUK PERANGKAT LUNAK
- SYSTEM SOFTWARE
  > PROGRAM UNTUK MELAYANI PROGRAM LAIN
  > BANYAK BERINTERAKSI DENGNA PERANGKAT LUNAK
- REAL TIME SOFTWARE
  > PERANGKAT LUNAK YANG MEMONITOR, MENGANALISA, MENGENDALIKAN SUATU KEJADIAN ATAU PERISTIWA YANG SEDANG TERJADI.
  > WAKTU TANGGAP (RESPONSE) SANGAT SINGKAT
